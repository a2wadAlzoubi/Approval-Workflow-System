​🚀 Approval Workflow System
​نظام متطور لإدارة سير العمل والاعتمادات (Approval Workflow) تم بناؤه باستخدام تقنيات .NET Core MVC واتباع أفضل الممارسات في هندسة البرمجيات لضمان القابلية للتوسع والصيانة.

​💡 فكرة المشروع (Project Concept)
​يهدف هذا النظام إلى أتمتة عمليات الاعتماد الورقية أو العشوائية داخل المؤسسات وتحويلها إلى مسار رقمي منظم. بدلاً من ضياع الطلبات في رسائل البريد الإلكتروني، يوفر النظام بيئة مركزية لإدارة أي طلب يحتاج إلى موافقة (مثل: طلبات الإجازات، شراء معدات، أو اعتماد مستندات).
​
🔄 دورة حياة الطلب في النظام:
​إنشاء الطلب: يقوم الموظف بإنشاء طلب وتحديد النوع والتفاصيل.

​تحديد المسار: بناءً على قواعد العمل (Business Rules)، يحدد النظام تلقائياً سلسلة المديرين المسؤولين عن الاعتماد.

​التفاعل والمراجعة: يصل إشعار للمدير المعني للموافقة، الرفض، أو طلب تعديلات.

​التتبع والشفافية: يمكن لمقدم الطلب رؤية "أين يقف الطلب الآن" ومن هو الشخص الذي يتوقف عنده الإجراء.

​الأرشفة: بمجرد اكتمال الموافقات، يُغلق الطلب ويُحفظ كمرجع تاريخي غير قابل للتلاعب.

​🎯 لماذا هذا المشروع؟ (Value Proposition)

​تقليل الوقت الضائع: التخلص من التأخير الناتج عن الانتظار اليدوي للموافقات.

​الشفافية (Transparency): معرفة المسؤول عن تعطيل أي إجراء بضغطة زر.

​دقة البيانات: فرض قيود (Validation) على الطلبات لضمان عدم تقديم بيانات خاطئة منذ البداية.

​المرونة: بفضل استخدام الـ DDD، يمكن إضافة قوانين عمل معقدة (مثل: "إذا كان المبلغ > 5000$ يحتاج موافقة المدير المالي") بسهولة دون تخريب الكود.

​🏗️ التصميم الهندسي (Architectural Patterns)

​يعتمد المشروع على بنية Clean Architecture لضمان فصل الاهتمامات (Separation of Concerns)، بالإضافة إلى الأنماط التالية:
​Domain-Driven Design (DDD): التركيز على منطق العمل الأساسي وفصله عن التفاصيل التقنية.

​CQRS Pattern: فصل عمليات القراءة (Queries) عن عمليات الكتابة (Commands) باستخدام مكتبة MediatR.

​Repository Pattern: لتجريد طبقة الوصول إلى البيانات.

​Dependency Injection: لإدارة دورة حياة الكائنات وتقليل الاعتماديات المباشرة.
​🛠️ التقنيات المستخدمة (Tech Stack)

​Framework: .NET 10.0 (ASP.NET Core MVC)
​Database: Entity Framework Core
​Communication: MediatR (لتحقيق نمط CQRS)
​Mapping: AutoMapper
​UI: Bootstrap, jQuery, Razor Pages
​📂 هيكلية المشروع (Project Structure)
​ينقسم المشروع إلى الطبقات التالية:
​Domain: تحتوي على الكيانات (Entities)، القواعد الأساسية (Value Objects)، وواجهات المستودعات (Repository Interfaces). "خالية من أي اعتماديات خارجية".
​Application: تحتوي على منطق العمل (Use Cases)، الـ DTOs، الـ Commands، والـ Queries (MediatR Handlers).
​Infrastructure: تحتوي على إعدادات قاعدة البيانات (Entity Framework)، وتطبيق المستودعات (Repositories)، وأي خدمات خارجية (مثل إرسال البريد الإلكتروني).
​WebUI (Presentation): طبقة العرض (ASP.NET Core MVC) التي تتعامل مع المستخدم النهائي.
​🔑 المميزات الرئيسية
​✅ إدارة طلبات الاعتماد: تقديم الطلبات وتتبع حالتها.
​✅ سلسلة الموافقات الديناميكية: إمكانية تحديد مسارات اعتماد مختلفة بناءً على نوع الطلب.
​✅ لوحة تحكم (Dashboard): لمتابعة الإحصائيات والطلبات المعلقة.
​✅ نظام صلاحيات متقدم: إدارة الأدوار والمستخدمين.
​🚀 كيفية التشغيل (Getting Started)
​المتطلبات المسبقة:
​Visual Studio 2022 أو VS Code.
​.NET SDK (الإصدار المستخدم في المشروع).
​SQL Server.
​خطوات التشغيل:
1.
git clone https://github.com/7ammoodeh/ApprovalWorkflowSystem.git
2.
تعديل سلسلة الاتصال (Connection String) في ملف appsettings.json داخل مشروع WebUI.
